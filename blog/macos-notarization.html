macOS 앱 공증(Notarization) 완전 가이드

2025.10.15 · 개발 이야기 · 약 9분

공증이 필요한 이유

macOS에는 Gatekeeper라는 보안 시스템이 있습니다. macOS 10.15 Catalina부터 Apple은 Mac App Store 외부에서 배포되는 모든 앱에 공증(Notarization)을 요구합니다. 공증되지 않은 앱을 실행하면 경고 대화상자가 표시되고, 일부 macOS 버전에서는 실행 자체가 차단됩니다.

공증이란 개발자가 앱을 Apple 서버에 제출하면, Apple이 악성 코드 검사를 수행하고 안전하다고 판단되면 티켓을 발행하는 과정입니다. 이 티켓이 있는 앱만 Gatekeeper 경고 없이 실행됩니다.

정리하면, macOS에서 앱을 배포하려면 세 가지가 필요합니다.

  1. 코드 서명: 개발자 신원 증명
  2. 공증: Apple의 악성 코드 검사 통과
  3. 스테이플링: 공증 티켓을 앱에 첨부 (오프라인 검증용)

준비물

1. Apple Developer 계정

Apple Developer Program에 등록해야 합니다. 연간 $99(약 13만 원)의 비용이 발생합니다. 무료 Apple ID만으로는 배포용 인증서를 발급받을 수 없습니다.

2. Developer ID Application 인증서

Mac App Store 외부 배포에 사용하는 인증서입니다. Apple Developer 웹사이트 > Certificates, Identifiers & Profiles에서 생성합니다. "Developer ID Application" 유형을 선택해야 합니다. "Mac Development"나 "Mac App Distribution"과는 다릅니다.

인증서 생성 절차는 다음과 같습니다.

  1. Mac의 키체인 접근 > 인증서 지원 > 인증 기관에서 인증서 요청에서 CSR(Certificate Signing Request) 생성
  2. Apple Developer 사이트에서 CSR을 업로드하여 인증서 발급
  3. 발급된 .cer 파일을 다운로드하여 더블 클릭으로 키체인에 설치

3. 앱 전용 비밀번호

공증 API 인증에 Apple ID 비밀번호를 직접 사용할 수 없습니다. 대신 앱 전용 비밀번호(App-Specific Password)를 생성합니다. appleid.apple.com > 보안 > 앱 전용 비밀번호에서 생성할 수 있습니다.

생성된 비밀번호는 xxxx-xxxx-xxxx-xxxx 형식입니다. 이 비밀번호는 한 번만 표시되므로 안전하게 보관해야 합니다.

4. Team ID

Apple Developer 계정의 Team ID입니다. developer.apple.com > 멤버십에서 확인할 수 있습니다. 10자리 영문 숫자 조합입니다.

코드 서명 과정

Tauri 프레임워크를 사용하는 경우, 빌드 시 자동으로 코드 서명이 이루어집니다. 환경 변수만 올바르게 설정하면 됩니다.

필요한 환경 변수는 다음과 같습니다.

  • APPLE_SIGNING_IDENTITY: 인증서 이름 (예: "Developer ID Application: Your Name (TEAMID)")
  • APPLE_ID: Apple 계정 이메일
  • APPLE_PASSWORD: 앱 전용 비밀번호
  • APPLE_TEAM_ID: Team ID

Tauri가 아닌 일반 앱의 경우, codesign 명령어로 직접 서명합니다.

codesign --deep --force --verify --verbose --sign "Developer ID Application: ..." YourApp.app

서명이 올바른지 확인하려면 아래 명령을 사용합니다.

codesign --verify --verbose=2 YourApp.app

공증 제출 (notarytool)

macOS 13부터는 notarytool이 공식 도구입니다. 이전의 altool은 더 이상 사용되지 않습니다.

공증 제출 명령은 다음과 같습니다.

xcrun notarytool submit YourApp.dmg --apple-id "email" --password "app-specific-password" --team-id "TEAMID" --wait

--wait 옵션을 추가하면 공증이 완료될 때까지 기다립니다. 완료 후 상태가 "Accepted"이면 성공입니다.

공증이 완료되면 스테이플링을 수행합니다.

xcrun stapler staple YourApp.dmg

스테이플링은 공증 티켓을 파일에 직접 첨부하는 과정입니다. 이렇게 하면 사용자가 오프라인에서도 공증 상태를 확인할 수 있습니다.

GitHub Actions에서 자동화

CI/CD 환경에서 공증을 자동화하는 것이 실무에서 가장 중요합니다. 매번 수동으로 서명하고 공증하는 것은 비현실적입니다.

시크릿 설정

GitHub 저장소의 Settings > Secrets and variables > Actions에 다음 시크릿을 등록합니다.

  • APPLE_CERTIFICATE: .p12 인증서 파일의 Base64 인코딩
  • APPLE_CERTIFICATE_PASSWORD: .p12 파일 비밀번호
  • APPLE_ID: Apple 계정 이메일
  • APPLE_PASSWORD: 앱 전용 비밀번호
  • APPLE_TEAM_ID: Team ID
  • APPLE_SIGNING_IDENTITY: 인증서 이름

인증서를 Base64로 변환

키체인에서 인증서와 개인 키를 .p12 파일로 내보낸 후, Base64로 인코딩합니다.

base64 -i certificate.p12 | pbcopy

이 값을 APPLE_CERTIFICATE 시크릿에 저장합니다.

워크플로우에서 인증서 설치

GitHub Actions 워크플로우에서 키체인을 생성하고 인증서를 설치합니다. Tauri의 공식 GitHub Action을 사용하면 이 과정이 대부분 자동화됩니다. 환경 변수에 시크릿 값을 전달하기만 하면 됩니다.

트러블슈팅

첫 공증에 26분 소요

OndamZip의 첫 공증은 무려 26분이 걸렸습니다. Apple 서버가 새로운 앱과 개발자를 처음 검사할 때 시간이 오래 걸리는 것으로 보입니다. 두 번째 공증부터는 12분 정도로 줄었고, 이후에는 5~8분 정도 걸립니다.

CI/CD 타임아웃을 넉넉하게 설정하는 것이 중요합니다. OndamZip은 GitHub Actions 타임아웃을 120분으로 설정했습니다. 기본 타임아웃(360분)을 사용해도 되지만, 명시적으로 설정하는 것이 좋습니다.

공증 실패: Invalid Signature

가장 흔한 실패 원인은 서명 문제입니다. 확인할 사항은 다음과 같습니다.

  • 인증서 유형이 "Developer ID Application"인지 확인 (다른 유형의 인증서는 배포용으로 사용 불가)
  • --deep 옵션으로 모든 번들 내부까지 서명했는지 확인
  • Hardened Runtime이 활성화되어 있는지 확인
  • entitlements 파일이 올바른지 확인

공증 실패: Contains Unsigned Binaries

앱 번들 내부의 모든 실행 파일과 동적 라이브러리가 서명되어야 합니다. 서드파티 라이브러리가 서명 없이 포함되면 공증이 거부됩니다. codesign --deep으로 전체를 재서명하면 해결되는 경우가 많습니다.

팁과 모범 사례

앱 전용 비밀번호 관리

앱 전용 비밀번호는 최대 25개까지 생성 가능합니다. CI/CD용, 로컬 개발용 등 용도별로 분리하여 생성하는 것을 권장합니다. 비밀번호가 노출되었거나 사용하지 않는 비밀번호는 즉시 폐기하세요.

인증서 갱신

Developer ID Application 인증서의 유효 기간은 5년입니다. 만료 전에 새 인증서를 발급받아야 합니다. 만료되면 기존에 서명한 앱은 계속 작동하지만, 새로운 빌드에 서명할 수 없습니다. 인증서를 갱신하면 GitHub 시크릿도 함께 업데이트해야 합니다.

로컬 테스트

CI에 올리기 전에 로컬에서 서명과 공증을 테스트하는 것이 좋습니다. CI에서 실패하면 디버깅이 어렵기 때문입니다. 로컬에서 성공한 설정을 그대로 CI 환경 변수로 옮기면 대부분 문제가 없습니다.

macOS 공증은 처음 설정할 때 복잡하지만, 한 번 자동화해두면 이후에는 신경 쓸 일이 거의 없습니다. 이 가이드가 도움이 되셨다면 OndamSoft의 다른 글도 확인해보세요.
