OndamZip 출시 - 왜 새 압축 프로그램을 만들었나

2025.01.15 · 5분 읽기

기존 압축 프로그램에 대한 불만

Windows에서 가장 많이 사용되는 무료 압축 프로그램이라 하면 7-Zip과 반디집을 떠올릴 것입니다. 둘 다 훌륭한 프로그램이지만, 개발자 입장에서 몇 가지 아쉬운 점이 있었습니다.

첫째, UI가 오래되었습니다. 7-Zip은 2000년대 초반 디자인을 거의 그대로 유지하고 있어, 최신 Windows 11의 깔끔한 인터페이스와 어울리지 않습니다. 반디집은 UI가 현대적이지만 무료 버전에 광고가 표시되는 점이 불편합니다.

둘째, 최신 압축 포맷 지원이 부족합니다. Facebook이 개발한 Zstandard(ZST)는 압축 속도와 비율 모두 뛰어나지만, 기존 프로그램에서의 지원이 제한적입니다. 특히 ZST 파일을 직접 생성하거나 TAR.ZST를 만드는 것은 쉽지 않았습니다.

셋째, 멀티스레드 활용이 아쉽습니다. 현대 CPU는 8코어, 16스레드 이상이 기본인데, 압축/해제 시 모든 코어를 효율적으로 활용하는 프로그램은 많지 않습니다.

OndamZip이 해결하려는 문제

이러한 불만을 해결하기 위해 OndamZip을 개발했습니다. 핵심 목표는 다음과 같습니다:

  • 현대적인 UI — Windows 11 디자인 언어에 맞춘 깔끔한 인터페이스
  • 광고 없음, 완전 무료 — 개인, 기업, 공공기관 모두 무료
  • 최신 포맷 지원 — ZIP, 7Z, RAR, TAR, GZ, ZST, XZ 등 주요 포맷 모두 지원
  • 멀티스레드 압축 — 모든 CPU 코어를 활용한 빠른 압축/해제
  • 실시간 진행률 — 대용량 파일 작업 시 정확한 진행 상황 표시

기술적 선택: Tauri + Rust

OndamZip은 Tauri 2.x 프레임워크로 개발했습니다. Electron 대신 Tauri를 선택한 이유는 명확합니다:

  • 설치 파일 크기 — Electron 앱은 보통 100MB 이상이지만, Tauri는 시스템의 WebView를 사용하므로 10MB 이하로 만들 수 있습니다
  • 메모리 사용량 — Chromium을 내장하지 않으므로 메모리 사용량이 훨씬 적습니다
  • Rust 백엔드 — 압축/해제 같은 CPU 집약적 작업에 Rust의 성능이 빛을 발합니다

프론트엔드는 React 19를 사용했고, 가상 스크롤링으로 수만 개의 파일이 포함된 압축 파일도 부드럽게 표시합니다.

멀티스레드 압축의 구현

OndamZip의 핵심 기술 중 하나는 멀티스레드 압축입니다. Rust의 rayon 라이브러리를 활용해 파일별 병렬 처리를 구현했습니다.

ZIP 포맷의 경우 파일 I/O를 병렬로 처리하고, TAR.ZST는 Zstandard의 네이티브 멀티스레드 모드를 활용합니다. 8코어 CPU 기준으로 단일 스레드 대비 3~5배 빠른 압축 속도를 달성했습니다.

실시간 진행률은 AtomicUsize와 AtomicU64를 사용해 스레드 간 안전하게 공유되며, 프론트엔드에서 200ms 간격으로 업데이트됩니다.

지원하는 포맷

OndamZip은 다양한 압축 포맷을 지원합니다:

포맷압축해제특징
ZIP✅✅가장 보편적, 호환성 최고
7Z❌✅높은 압축률
RAR❌✅게임/멀티미디어에서 많이 사용
TAR.GZ✅✅Linux 표준
TAR.ZST✅✅최신, 빠른 속도 + 높은 압축률
TAR.XZ✅✅최고 압축률
TAR.BZ2✅✅Unix 전통 포맷
분할(.001)❌✅대용량 파일 분할

macOS 지원

OndamZip은 Windows뿐 아니라 macOS(Apple Silicon)도 지원합니다. Apple의 코드서명과 공증(Notarization)을 완료해 macOS에서도 경고 없이 설치할 수 있습니다. macOS 버전은 DMG로 배포되며, __MACOSX 메타데이터와 .DS_Store 파일을 자동으로 필터링합니다.

앞으로의 계획

OndamZip은 계속 발전하고 있습니다. 향후 계획은 다음과 같습니다:

  • 7Z, RAR 압축 생성 지원
  • 드래그 앤 드롭 개선
  • 탐색기 컨텍스트 메뉴 통합 강화
  • 암호화된 ZIP/7Z 생성
  • Linux 지원
